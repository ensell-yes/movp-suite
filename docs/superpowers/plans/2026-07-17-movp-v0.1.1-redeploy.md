# MOVP Suite v0.1.1 — Redeploy to close the 170→176 agent-surface gap (movp-prod)

> **For agentic workers:** This is an **operator-executed deployment runbook**, NOT a subagent-auto-executed plan. It applies schema to a live production database and redeploys production edge functions + the frontend Worker — outward-facing, hard-to-reverse actions. A human runs each task and confirms its gate before the next. Steps use checkbox (`- [ ]`) syntax for tracking. Do not auto-execute end-to-end.

**Goal:** Bring the deployed `movp-prod` stack up from the v0.1.0 build (live MCP registry = **170** tools, with 7 forbidden `.create` tools exposed and no DB write-guard backstop) to current `main` (v0.1.1, registry = **176** tools, forbidden creates removed, DB grants hardened), so the live surface matches the published 0.1.1 agent contract the other developer is following.

**Root cause (why this deploy exists):** The 176-tool contract and its DB write-guards landed in commit `6723101` ("feat: harden agent interfaces and export contracts") and migration `20260717000001_agent_surface_write_guards.sql` — both **after** the v0.1.0 deploy tag `34f5be5` (fix dated 2026-07-17 11:18; tagged commit dated 05:12). `git merge-base --is-ancestor 6723101 v0.1.0` returns non-zero (NOT an ancestor). The current `main` code is correct — the exporter produces 176; only the *deployment* is stale. Arithmetic: `176 − 13 generic .update tools (not yet added) + 7 forbidden .create tools (not yet removed) = 170`.

**Architecture:** The fix is two independent layers, and this runbook ships **both**:
1. **Registry layer** — redeploying the `mcp` edge function (and its siblings that share `_shared`/`deno.json`) publishes the corrected tool set: adds the 13 generic `.update` tools (incl. `campaign.update`), removes the 7 `.create` tools on `write-mode-none` collections.
2. **Database layer** — migration `20260717000001_agent_surface_write_guards.sql` `revoke insert,update,delete … from authenticated` on `segment_membership`, `segment_snapshot`, `segment_recompute_run`, and tightens `campaign_metric` to insert-only. This is the backstop so a stale/cached client cannot still write the forbidden rows.

Shipping only one layer leaves a hole; this plan sequences the DB layer first (safety-critical), then the registry, then verifies both are consistent.

**Tech Stack:** Supabase Cloud (Postgres/GoTrue/Deno edge), Supabase CLI `2.109.1`, Deno `2.9.2`, Cloudflare Workers (Norcal Crew), `wrangler ^4`, Astro 6 + `@astrojs/cloudflare`. All infra already provisioned at v0.1.0.

**Prior art:** `docs/superpowers/plans/2026-07-16-movp-v0.1-supabase-cloudflare-deploy.md` (the full v0.1.0 runbook). This plan reuses its idioms (ref-safety, token export, `db push`, `functions deploy`, frontend build/deploy) and **skips** every provisioning task in it.

## Risks & notes

- **Deploying only one layer (top risk).** The registry redeploy (Task 3) and the DB migration (Task 2) each close half the gap. Task 2 MUST precede Task 3's verification: apply the `revoke` first so that at no point is a forbidden `.create` both advertised *and* writable. If you must split the deploy, do the migration first — it is the safety backstop.
- **Stale function cache.** After `functions deploy`, the edge may briefly serve the prior build. Task 3 Step 3 is the guard — if it reports 170, wait and re-check before assuming failure.
- **Wrong-project push (catastrophic, irreversible).** `movp-prod` = `poocqnzsrwipbweeacbd`; norcalOS look-alike = `fmnwipusmtloximgtzww`. Task 1 Step 5 verifies the name before `db push`. Never skip it.
- **Do-not-recreate infra.** The workspace, PAT, secrets, crons, R2 bucket, and Auth config are live from v0.1.0. This plan reuses them read-only. A stray `create_workspace`/PAT-mint/`vault.create_secret` is silent drift — see Global Constraints.
- **Migration count drift.** If Task 1 Step 6 shows anything other than exactly the two expected files, the target or baseline is wrong — do not push. If it shows zero, the DB is already at 0.1.1 and only the edge functions are stale (skip Task 2).
- **npm publish deferred.** This is a service redeploy + git tag only, consistent with v0.1.0. `@movp/*` package publication is out of scope.
