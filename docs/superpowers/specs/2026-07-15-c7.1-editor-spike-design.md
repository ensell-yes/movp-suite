# C7.1 — Editor Dependency Spike (Design)

**Date:** 2026-07-15
**Phase:** Stage C7 — Inline Editing & Content Delivery (first task)
**Status:** design, pending plan
**Precondition:** C1 merged (OSS packaging). Does not depend on C7.2–C7.7.

## 1. Goal & exit criterion

Decide **BlockNote vs TipTap** as the editor for Phase C7, backed by evidence. A
candidate is acceptable only if it can round-trip a MOVP `content_revision.data`
record **hash-stably through the create → edit → publish lifecycle**, render the
published result read-only, stay within the client/server bundle boundary, and clear
the license bar — measured head-to-head, at attributable cost.

The spike is **throwaway**. Its durable outputs are:

1. A decision report (`docs/superpowers/specs/2026-07-15-c7.1-editor-spike-report.md`)
   with the comparison matrix, license findings, bundle numbers, and a recommendation.
2. The **reproducible harness source + one nested lockfile**, kept committed through
   C7.1 and removed in C7.2 after the winning adapter's contract test migrates into
   `@movp/editor-sdk` (see §10).

The **permissive-vs-MPL production license call stays with the human operator**; the
spike produces the data, not the verdict.

## 2. Background — the data the editor must round-trip

Verified against the codebase (cited so the implementer trusts it, doesn't re-derive):

- `content_revision.data` is a free-form `f.json()` blob
  (`packages/core-schema/src/collections/content_revision.ts`).
- The blob is validated against a per-content-type `field_schema`, canonicalized
  (outer keys sorted), and SHA-256 hashed for conflict detection
  (`packages/domain/src/content.ts:159–173`, `sha256Hex` uses `crypto.subtle`).
- The field type is spelled **`richtext`** (lowercase) and is validated as
  **`z.string()`** — an editor's rich content is stored as an **opaque string**,
  not a nested object (`packages/domain/src/content.ts:33,63–72`).
- `ContentService` exposes `create` (`create_content_with_revision`), `update`
  (`update_content`, with `p_expected_revision_id` → `content_update_conflict`),
  `publish` (`publish_content`), and `getPublished` — which reads
  `content_item.published_revision_id`, then that row from `content_revision`, returning
  `{ item, revision }` (`packages/domain/src/content.ts:207,224,316,328`).
- `canonicalize` and `sha256Hex` are **not exported**; only `makeContentService`
  is (`:110`). The oracle (§4) must go through it.
- `DomainCtx` requires `{ db: SupabaseClient; userId: string }` plus optional
  `accessToken`, `assetsFnUrl` (`packages/domain/src/types.ts:38–42`).

## 3. Scope

**In scope:** prove idempotent round-trip AND the create/edit/publish lifecycle plus
read-only published render, against a multi-field fixture; measure license, bundle,
boundary, a11y for both candidates; recommend one.

**Out of scope (deferred to C7.2+):** the production `@movp/editor-sdk`, the in-place
overlay, realtime presence, delivery artifacts (sitemap/JSON-LD/etc.), and the real
conflict UX. The spike builds only what a decision requires. It uses **no live
database** — the lifecycle runs against the stateful oracle in §4.

## 4. The lifecycle oracle — a real `SupabaseClient` with a stateful fake `fetch`

`DomainCtx.db` is a concrete `SupabaseClient` (`types.ts:39`), so a two-method fake
cannot be passed to `makeContentService` without a type escape. Instead the oracle
constructs a real client whose network layer is a **stateful** fake `fetch` that
tracks revisions and the published pointer. This needs no `any` and no cast, exercises
real PostgREST request serialization, and lets the plan-mandated create → edit →
publish lifecycle run end-to-end without a live database.

```ts
// @spike/oracle — Node-only. NEVER imported from a browser/client path (see §6).
import { createClient } from '@supabase/supabase-js'
import { makeContentService } from '@movp/domain'
import { FIXTURE_FIELD_SCHEMA } from '@spike/fixture'

export interface RpcCapture {
  rpc: string
  p_data?: Record<string, unknown>
  p_content_hash?: string
  p_expected_revision_id?: string | null
}

export interface LifecycleOracle {
  service: ReturnType<typeof makeContentService>
  captures: RpcCapture[] // ordered record of every RPC the service issued
  currentRevisionId(): string
  publishedRevisionId(): string | null
  /** Sabotage hook (§5.4 step 6): force the published pointer to a chosen revision id. */
  forcePublishedRevision(id: string): void
}

function isRecord(value: unknown): value is Record<string, unknown> {
  return typeof value === 'object' && value !== null && !Array.isArray(value)
}

export function parseRpcBody(init: RequestInit | undefined): Record<string, unknown> {
  if (!init?.body) return {}
  let parsed: unknown
  try {
    parsed = JSON.parse(String(init.body))
  } catch {
    throw new Error('oracle_invalid_rpc_body')
  }
  if (!isRecord(parsed)) throw new Error('oracle_invalid_rpc_body')
  return parsed
}

export function requireRevisionFields(body: Record<string, unknown>): {
  data: Record<string, unknown>
  hash: string
} {
  if (!isRecord(body.p_data) || typeof body.p_content_hash !== 'string') {
    throw new Error('oracle_invalid_rpc_body')
  }
  return { data: body.p_data, hash: body.p_content_hash }
}

export function makeLifecycleOracle(): LifecycleOracle {
  const captures: RpcCapture[] = []
  // Revision-data store so getPublished() can read back the real published body.
  const revisions = new Map<string, { data: Record<string, unknown>; content_hash: string }>()
  let revSeq = 0
  const item: {
    id: string
    content_type_id: string
    current_revision_id: string
    published_revision_id: string | null
  } = {
    id: 'item',
    content_type_id: 'ct',
    current_revision_id: '',
    published_revision_id: null,
  }

  const json = (obj: unknown, ct = 'application/json'): Response =>
    new Response(JSON.stringify(obj), { status: 200, headers: { 'content-type': ct } })

  const fakeFetch: typeof fetch = async (input, init) => {
    const url = typeof input === 'string' ? input : input.toString()
    const body = parseRpcBody(init)

    // loadType(): GET .../content_type?select=field_schema  (maybeSingle)
    if (url.includes('/rest/v1/content_type')) {
      return json({ field_schema: FIXTURE_FIELD_SCHEMA }, 'application/vnd.pgrst.object+json')
    }
    // getPublished() step 2: GET .../content_revision?...&id=eq.<rev>  (maybeSingle)
    if (url.includes('/rest/v1/content_revision')) {
      const id = decodeURIComponent(/id=eq\.([^&]+)/.exec(url)?.[1] ?? '')
      const rev = revisions.get(id)
      return json(rev ? { id, ...rev } : null, 'application/vnd.pgrst.object+json')
    }
    // content_item: serves both itemTypeId() (select=content_type_id) and
    // getPublished() (select=*). The full row satisfies either projection.
    if (url.includes('/rest/v1/content_item')) {
      return json({ ...item }, 'application/vnd.pgrst.object+json')
    }
    if (url.includes('/rpc/create_content_with_revision')) {
      const rev = `r${++revSeq}`
      const fields = requireRevisionFields(body)
      const cap: RpcCapture = {
        rpc: 'create_content_with_revision',
        p_data: fields.data,
        p_content_hash: fields.hash,
      }
      captures.push(cap)
      revisions.set(rev, { data: fields.data, content_hash: fields.hash })
      item.current_revision_id = rev
      return json({ id: item.id, current_revision_id: rev })
    }
    if (url.includes('/rpc/update_content')) {
      const rev = `r${++revSeq}`
      const fields = requireRevisionFields(body)
      const expectedRevisionId = body.p_expected_revision_id
      if (expectedRevisionId !== null && typeof expectedRevisionId !== 'string') {
        throw new Error('oracle_invalid_rpc_body')
      }
      const cap: RpcCapture = {
        rpc: 'update_content',
        p_data: fields.data,
        p_content_hash: fields.hash,
        p_expected_revision_id: expectedRevisionId,
      }
      captures.push(cap)
      revisions.set(rev, { data: fields.data, content_hash: fields.hash })
      item.current_revision_id = rev
      return json({ id: item.id, current_revision_id: rev })
    }
    if (url.includes('/rpc/publish_content')) {
      item.published_revision_id = item.current_revision_id
      captures.push({ rpc: 'publish_content' })
      return json({ id: item.id, published_revision_id: item.published_revision_id })
    }
    throw new Error(`oracle: unexpected request ${url}`)
  }

  const db = createClient('http://spike.invalid', 'stub-anon-key', {
    global: { fetch: fakeFetch },
  })
  const service = makeContentService({ db, userId: 'spike-user' })
  return {
    service,
    captures,
    currentRevisionId: () => item.current_revision_id,
    publishedRevisionId: () => item.published_revision_id,
    forcePublishedRevision: (id: string) => {
      item.published_revision_id = id
    },
  }
}

/** Standalone production hash for one record (fresh oracle, create path only). */
export async function hashOnce(data: Record<string, unknown>): Promise<string> {
  const o = makeLifecycleOracle()
  await o.service.create({ workspaceId: 'ws', contentTypeId: 'ct', slug: 's', data })
  const cap = o.captures.find((c) => c.rpc === 'create_content_with_revision')
  if (!cap?.p_content_hash) throw new Error('hashOnce: no create hash captured')
  return cap.p_content_hash
}
```

Every captured `p_content_hash` is the **genuine production hash**, because the
service runs the real `fieldSchemaToZod` → `canonicalize` → `sha256Hex`.
Malformed or structurally incomplete fake-transport bodies fail immediately with the
stable `oracle_invalid_rpc_body` diagnostic; tests cover invalid JSON, non-object JSON,
and missing/wrong-typed revision fields.

## 5. Adapter contract, canonical algorithm, and the lifecycle proof

### 5.1 Adapter (one per candidate)

```ts
export interface EditorAdapter<Doc> {
  /** stored richtext string -> editor document */
  decode(body: string): Doc
  /** editor document -> stored richtext string, via the §5.2 canonical algorithm */
  encode(doc: Doc): string
}
export const INNER_CANONICAL_VERSION = 1 // bump if §5.2 changes
```

### 5.2 Canonical inner-JSON algorithm (normative)

Because `richtext` is an opaque string and production `canonicalize` sorts only the
**outer** `data` keys, idempotency reduces to: **is the stored string byte-stable
across load→save?** `encode` therefore serializes the editor document through exactly
this algorithm — the byte contract, not a hand-wave:

- Accept only JSON values: plain object, array, string, boolean, `null`, finite number.
- **Throw** on `undefined`, `bigint`, `NaN`, `±Infinity`, or a non-plain object.
- Recursively sort object keys lexicographically by UTF-16 code unit.
- Preserve array order.
- Emit compact `JSON.stringify` output (no spaces or newlines).

**Block-ID policy:** **preserve the editor's explicit native IDs by default.** Do NOT
derive IDs from content — content-derived IDs get rewritten by an ordinary text edit
and would damage future block identity and collaboration. Any ID stripping or
normalization is an **experimental result recorded in the report**, never the default.

### 5.3 Fixture (multi-field — the editor occupies ONE field of a record)

`FIXTURE_FIELD_SCHEMA = [{name:'title',type:'text'}, {name:'body',type:'richtext'}, {name:'meta',type:'json'}]`

### 5.4 Lifecycle proof (per candidate) — mirrors the plan's create/edit/publish

Playwright serves a Vite page and mounts the real editor; Node holds the oracle.
`decode`/`encode` bridge the two. Steps:

1. `seedData = { title, body: adapter.encode(seedDoc), meta }`; `H0 = await hashOnce(seedData)`.
2. **Zero-edit idempotency:** Playwright mounts the editor, loads `adapter.decode(body)`,
   makes no edit, returns the live document; `H1 = await hashOnce({ ...seedData, body: adapter.encode(returnedDoc) })`.
   **PASS = `H1 === H0`.**
3. **Editor-driven edit (NOT hand-built in Node):** in the mounted editor, select an existing
   word and toggle bold via a real user action (Playwright `dblclick` + `ControlOrMeta+b`, or a
   `boldWord()` mount command that calls the editor's own bold command — no new/random block IDs).
   Read the result with `window.__spike.serialize()`; `editedBody` is that string;
   `H2 = await hashOnce({ ...seedData, body: editedBody })`. Assert `H2 !== H0` **and** the JSON
   diff vs the seed body is exactly the one bolded word. A variant that skips the bold action must
   leave `H2 === H0` (so the assertion is proven meaningful).
4. **Lifecycle (single ordered oracle):** `o = makeLifecycleOracle()`.
   - `await o.service.create({ workspaceId:'ws', contentTypeId:'ct', slug:'s', data: seedData })`
     → capture[0] is `create_content_with_revision`; `r1 = o.currentRevisionId()`.
   - `await o.service.update({ itemId:'item', expectedRevisionId: r1, data: editedData })`
     → capture[1] is `update_content`; assert `p_expected_revision_id === r1` and
     `p_content_hash === H2`; `r2 = o.currentRevisionId()`.
   - `await o.service.publish({ itemId:'item' })` → capture[2] is `publish_content`;
     assert `o.publishedRevisionId() === r2`.
   - Assert the **exact RPC route order** `[create, update, publish]` from `o.captures`.
5. **Read-only published render via the real delivery path:**
   `const published = await o.service.getPublished('item')`. Assert `published` is
   non-null and `published.revision.id === r2`; structurally narrow
   `published.revision.data` to a record and require a string `body`, failing otherwise.
   Mount the editor **read-only** with `adapter.decode(body)` and compare the rendered
   heading/list/inline-formatting to a **semantic golden** (DOM roles + text, not pixels).
   This proves create → edit → publish → **delivery read**, not a hand-fed body.
6. **Sabotage gate (per sabotage-test-every-gate):** `o.forcePublishedRevision(r1)` (the
   pre-edit revision), re-run `getPublished → decode → render`, and require the golden
   comparison to fail red — proving the test is wired through the real published pointer,
   so a broken `published_revision_id` read cannot pass silently.

Vitest is used **only** for pure oracle/fixture unit tests (`hashOnce` of a fixture equals
a pinned expected hash; `encode(decode(seed)) === seed`; reordered inner keys canonicalize
identically). No jsdom/happy-dom, no conditional headless fallback: **Playwright mounts,
Node hashes.**

## 6. Boundary discipline

- **`@spike/fixture`** is dependency-free and browser-safe (pure data + adapter types).
  Candidate client pages may import it.
- **`@spike/oracle`** imports `@movp/domain` and is **Node-only**; no browser/client
  module may import it.
- **Exact source boundary:** every browser-reachable candidate module, including the Vite
  entry, lives under that candidate's `src/browser/`; Node-only Playwright/Vitest files
  live under `test/`. A gate fails if the Vite entry is outside `src/browser/` or a
  browser module uses a relative import that escapes that root.
- **Source gate:** grep each candidate's `src/browser/` for `@movp/domain`, `@movp/auth`,
  `@supabase`, `packages/domain`, or `@spike/oracle` → any hit fails. A legitimate Node
  test may import `@spike/oracle`; the same import under `src/browser/` must fail.
  (`@spike/oracle` is listed so the §6 sabotage import trips the source gate **and** the
  module-id gate independently, not just the latter.)
- **Module-graph gate (real artifact, not a grep over minified output):** a small
  Vite/Rollup plugin uses `this.getModuleIds()` in `generateBundle` to emit a sorted
  `module-ids.json`. The gate fails if that artifact is missing or empty, then fails on
  any forbidden module id within it. Tree-shaking cannot hide a leak from `getModuleIds`.
- **Sabotage test:** add a temporary client import of `@spike/oracle` and confirm **both**
  the source gate and the module-id gate go red; then remove it.

### 6a. Guarded file I/O (untrusted-io rule)

Every gate/report script reads files (working-tree source, built `dist/`, `.report/`). Per the
untrusted-io-and-resource-bounds rule, all reads go through one shared `scripts/lib/safe-io.mjs`:
`lstat` before any read (reject symlinks and non-regular files — the source walker traverses the
working tree, where a symlinked file could escape the project), enforce a per-file and total-byte
cap before buffering, parse persisted JSON behind an `unknown` type-guard, and log **path + a
stable reason only** — never file contents. A symlinked, oversized, or malformed input must fail
the gate loudly without reading or echoing victim bytes.

## 7. Fair, attributable comparison

- **Equivalent UI surfaces.** Both candidates are React and implement one **common
  feature manifest**: paragraph, h1–h3, bold, italic, bullet list, ordered list,
  undo/redo, a toolbar, and keyboard shortcuts.
  - BlockNote: `@blocknote/core` + `@blocknote/react` + `@blocknote/mantine` (default UI).
  - TipTap: `@tiptap/react` + `@tiptap/pm` + `@tiptap/starter-kit` + a minimal matching toolbar.
- **Isolation.** Nested workspace `spikes/editor/` with **one** `spikes/editor/pnpm-lock.yaml`
  and members `@spike/{fixture,oracle,blocknote,tiptap}`. Editor deps live only in each
  candidate package, so `pnpm licenses` and bundle size are attributable per candidate.
  All deps **exact-pinned**.
- **Bundle metric (trimmed to go/no-go essentials).** Each candidate's Vite build reports
  its **total JS + CSS, raw and gzip**, measured via the shared guarded reader (§6a).
  BlockNote's total is its full default Mantine UI; TipTap's total is its minimal shared
  toolbar, so the report also records TipTap's **toolbar LOC** as the parity-cost signal.
  Initial-vs-lazy chunk split and a separate common-subset build are **dropped**: these
  islands have no dynamic imports, so the split is always zero, and a go/no-go decision does
  not need it. A candidate's bundle number is accepted only if its lifecycle + a11y results
  are `pass` (§12.2).

## 8. License as a compliance gate

Per candidate, produce **two** reports:

- `pnpm licenses list --prod --json` — the **production** decision input.
- A **full** report including dev/harness deps — repository compliance (note:
  `@axe-core/playwright` is itself MPL-2.0).

Machine checks (both fail-closed on any missing/unknown license):

- Separate **prod** and **dev** SPDX allowlists.
- Assert **no `@blocknote/xl-*`, GPL, AGPL, or proprietary** package entered the
  **production** graph (BlockNote's XL tier is GPL-3.0/commercial; `@blocknote/react`
  is MPL-2.0; TipTap OSS is MIT — the spike verifies, does not assert).
- Capture exact versions, the nested lockfile digest, and required notice texts.

## 9. A11y gate (trimmed to go/no-go essentials)

Both candidates render one shared toolbar (Bold, H1, Bullet List, Undo, Redo); each control is
a `<button>` with an accessible name. The check (Playwright + `@axe-core/playwright`) asserts:

- **axe after initial load:** zero `violations` **and** zero unresolved `incomplete` results.
- **focus lands on the editor:** after one `Tab`, `document.activeElement` is the editable
  region or a named toolbar button — never `<body>`. (A non-null active element alone is NOT a
  pass — browsers always have one.)
- **toolbar accessible names:** each toolbar control is reachable by role + name.

**Dropped for the go/no-go spike** (deferred to C7.2): the full focus-order/restoration matrix
and per-menu axe sweeps — they prove production editor UX, not editor viability.

## 12. Decision gate (what "PASS" means for the spike)

A candidate **passes** iff: (a) zero-edit `H1==H0` and an edit **driven through the mounted
editor** (not hand-built in Node) gives `H2!=H0` with an exact diff; (b) the create→update→publish RPC order and revision pointers assert
as in §5.4 and the read-only render matches the golden (and the r1 sabotage fails red);
(c) boundary source **and** module-id gates are empty (and the sabotage import goes red);
(d) a11y (axe + behavioral) is clean; (e) the prod license graph is free of
GPL/AGPL/proprietary/`@blocknote/xl-*`. The report then recommends one candidate, carrying
the license posture (permissive vs MPL) forward for the human call formalized in §12.1.

### 12.1 Terminal license decision (report schema — makes C7.1 complete)

The recommendation alone does not finish C7.1: the goal is a *decision*, and the
permissive-vs-MPL call is the operator's. The report MUST record a discriminated terminal
choice, and a report-validation gate enforces it:

```ts
type LicenseDecision =
  | { kind: 'accept_mpl' }        // BlockNote (MPL-2.0 prod deps) is permitted to win
  | { kind: 'permissive_only' }   // any MPL/copyleft in the prod graph disqualifies

interface SpikeResult {
  licenseDecision: LicenseDecision
  selectedCandidate: 'blocknote' | 'tiptap'
}
```

- **C7.1 is INCOMPLETE** until both `licenseDecision` and `selectedCandidate` are recorded.
- **Consistency gate (fail-closed):** `permissive_only` ⇒ `selectedCandidate` must be a
  candidate whose **prod** license graph is permissive-only. So `permissive_only` +
  `selectedCandidate: 'blocknote'` (whose prod graph carries MPL-2.0) FAILS validation;
  `permissive_only` selects TipTap only if TipTap also passes §12 (a)–(d).
- **Technical-pass gate (fail-closed):** `selectedCandidate` MUST have passed every
  applicable §12 candidate gate. Selecting a candidate that failed hash stability,
  lifecycle/delivery, boundary, a11y, or production-license validation FAILS even when
  the `licenseDecision` would otherwise permit it. If neither candidate passes, no
  selection may be recorded and C7.1 stops for redesign.
- Report validation fails if either field is absent, or if the pair is inconsistent with
  the measured per-candidate results or license graphs. A regression fixture selects a
  BlockNote result that failed idempotency under `accept_mpl` and must fail validation.

### 12.2 CandidateResult (the machine shape §12.1 validates)

Each candidate emits this validated object; a field is written only after its test's assertions
pass (the clean-run deletes `.report/` first, so a stale file cannot false-pass). `report.mjs`
rejects a `selectedCandidate` with any boolean `false`, `license !== 'pass'`, or a missing field,
and generates the human report (§10) from these results rather than hand-transcription:

```ts
interface CandidateResult {
  schemaVersion: 1
  candidate: 'blocknote' | 'tiptap'
  resolvedVersions: Record<string, string> // exact installed editor deps
  idempotent: boolean       // H1 === H0 (zero-edit round-trip through the mounted editor)
  exactEdit: boolean        // H2 !== H0; edit driven THROUGH the mounted editor; diff = bolded word only
  lifecycleOrder: boolean   // captures === [create, update, publish]; update hash === H2
  publishedRead: boolean    // getPublished returns r2 and renders read-only
  staleSabotage: boolean    // forcePublishedRevision(r1) yields the pre-edit body
  boundary: boolean         // source + module-id gates clean
  a11y: boolean             // §9
  license: 'pass' | 'fail'  // prod SPDX gate
  blockIdPreserved: boolean // native block IDs survived the zero-edit round-trip (recorded finding)
  bundle: { jsRaw: number; jsGzip: number; cssRaw: number; cssGzip: number }
  toolbarLoc?: number       // TipTap parity-cost signal (§7)
}
```
